# Doripe Instagram 콘텐츠 자동화 설계

## 목적

Doripe의 Instagram 콘텐츠 제작을 매일 반복 가능한 자동 작업으로 만든다. 초기에 Figma 디자인과 캡션 구조를 확정한 뒤에는 시스템이 소재 조사, 사진 수집, 문구 작성, Figma 슬롯 교체, 이미지 내보내기와 결과 정리까지 수행한다. 사용자는 완성본을 최종 확인하고 직접 게시한다.

## 확정 범위

- 매일 완성본을 최대 2개 생성한다.
- 별도 관리 사이트를 만들지 않고 Codex와 Figma를 사용한다.
- 디자인은 초기에만 확정하며 이후에는 사진과 문구 슬롯만 변경한다.
- 콘텐츠 유형은 `PLACE/EVENT`, `COLLECTION`, `ROUTE` 세 가지다.
- 세 유형 모두 단일 이미지가 아니라 정보형 캐러셀 피드로 제작한다.
- 사진은 웹에서 수집한 실제 사진만 사용하며 AI 이미지는 사용하지 않는다.
- 장소, 코스, 행사는 대한민국 국내 콘텐츠만 다룬다. 후보마다 `KR` 국가 코드와 국내임을 확인한 공식 출처를 기록하며, 해외 후보는 점수 계산 전에 차단한다.
- 재사용 허용이 확인된 사진을 우선한다.
- 일반 웹 사진을 사용하면 출처명, 원문 URL과 권리 상태를 반드시 기록한다.
- Instagram 자동 게시는 하지 않는다.

## 비목표

- Instagram 계정 로그인과 자동 게시
- 콘텐츠 관리용 대시보드
- 매 실행마다 새로운 디자인 제작
- AI 장소 사진 생성
- 사진 사용 권리의 법적 보장

## 운영 구조

### 1. Template Pack

초기 한 번만 확정하는 자산이다.

- Figma 원본 파일
- 세 가지 콘텐츠 템플릿
- 템플릿별 이미지·문구 슬롯 이름
- PNG 내보내기 설정
- 캡션 구조
- Doripe 브랜드 문구와 CTA 규칙

디자인 레이어는 잠그고 자동화가 수정할 수 있는 레이어만 다음과 같이 이름을 붙인다.

- `slot:title`
- `slot:subtitle`
- `slot:photo:01`, `slot:photo:02` ...
- `slot:place:01`, `slot:place:02` ...
- `slot:body:01`, `slot:body:02` ...
- `slot:info:location`
- `slot:info:date`
- `slot:cta`
- `slot:credit`

자동화는 `slot:*` 레이어만 변경한다. 배경, 로고, 서체, 색상, 위치와 장식은 변경하지 않는다.

### 2. Content Radar

실행할 때마다 장소, 코스, 전시, 팝업, 시즌 행사와 주제형 장소 묶음을 조사한다.

자료 확인 순서는 다음과 같다.

1. 행사 주최자·장소·기관의 공식 사이트와 공식 계정
2. 신뢰 가능한 뉴스·매거진
3. 블로그·SNS와 기타 웹 자료

후보는 다음 기준으로 100점 만점 평가한다.

- 친구에게 보내고 싶은 공유 가능성: 20점
- 나중에 다시 볼 저장 가치: 15점
- Doripe 브랜드 적합도: 15점
- 지금 소개할 이유와 시의성: 15점
- 사진 품질과 수량: 10점
- Doripe만의 편집·관점을 더할 가능성: 10점
- 장소·기간 등 사실 정보 완성도: 10점
- 사진 재사용 허용 여부: 5점

점수가 높은 후보부터 선택하되 한 콘텐츠 유형에만 몰리지 않도록 최근 7일 제작 이력을 함께 본다.

### 2.1 Feed Reach Strategy

조회수 최적화의 1차 목표는 단순 좋아요가 아니라 `친구에게 보내기`, `저장`, `게시물에 머문 시간`을 늘리는 것이다. Instagram은 Feed에서 머문 시간, 댓글, 좋아요, 공유와 프로필 방문 가능성을 예측하며, 비팔로워 도달에는 보내기 비율이 상대적으로 더 중요하다고 설명한다.

Doripe 콘텐츠에는 다음 원칙을 적용한다.

- 첫 장에서 누구에게 어떤 도움이 되는지 한 문장으로 약속한다.
- `이번 주말`, `7월까지만`, `비 오는 날`, `성수 3시간 코스`처럼 시점·상황·지역을 구체적으로 쓴다.
- 두 번째 장부터 바로 장소·행사·코스 정보를 제공하고 긴 도입은 쓰지 않는다.
- 각 장은 다음 장을 볼 이유가 있도록 정보 순서와 흐름을 만든다.
- 저장 가치가 있는 날짜·주소·동선·예약 정보·추천 이유를 포함한다.
- 친구와 함께 갈 콘텐츠에는 `같이 갈 친구에게 보내기`, 나중에 필요한 정보형 콘텐츠에는 `지금 저장하기` 중 하나만 CTA로 선택한다.
- 모든 게시물에 Instagram 위치 태그로 입력할 장소를 `review.txt`에 제안한다.
- 캡션 첫 부분에 지역, 장소 유형, 상황 키워드를 자연스럽게 넣는다.
- 관련 해시태그는 캡션에만 넣고 의미 없는 대량 해시태그는 사용하지 않는다.

### 3. Source Collector

선정된 후보의 사실 정보와 웹 사진을 모은다.

사진별로 다음 정보를 기록한다.

- 원본 파일명과 저장 파일명
- 사진이 게시된 페이지의 URL
- 출처 또는 계정명
- 확인 가능한 저작자·기관명
- 재사용 허용 상태: `confirmed`, `not_found`, `restricted`
- 얼굴·차량번호 등 개인정보 확인 메모

`confirmed` 사진을 먼저 사용한다. 적절한 사진이 부족하면 `not_found` 사진을 사용할 수 있지만 결과물의 `review.txt`에 권리 미확인 상태를 표시한다. `restricted` 사진은 사용하지 않는다.

이 기록은 사용 허가를 대신하지 않으며 사용자가 게시 전에 최종 판단한다.

### 4. Content Composer

확인된 자료만 사용해 표지 제목, 장별 문구와 캡션을 작성한다. 하나의 게시물에는 하나의 핵심 메시지와 하나의 CTA만 둔다.

캡션은 다음 순서를 따른다.

1. 관심을 끄는 첫 문장
2. 장소·행사·코스를 소개하는 본문
3. 장소명, 위치, 운영 기간과 필요한 방문 정보
4. 사진과 정보 출처
5. 저장·공유·앱 확인 중 하나의 CTA

확인되지 않은 운영 시간, 가격, 행사 기간과 기능을 추정해서 쓰지 않는다.

표지 문구는 다음 조건을 만족해야 한다.

- 한 게시물에 한 가지 약속만 한다.
- 장소명만 적지 않고 사용자가 얻는 이익이나 지금 봐야 할 이유를 적는다.
- 자극적인 과장보다 구체적인 지역·상황·기간을 사용한다.
- 본문이 약속을 실제로 충족하지 못하는 낚시성 제목은 금지한다.

### 5. Figma Filler

콘텐츠 유형에 맞는 템플릿을 복제하고 `slot:*` 레이어에 사진과 문구를 넣는다.

템플릿 구성은 다음과 같다.

#### PLACE/EVENT

- 표지
- 핵심 사진과 소개
- 장소 또는 행사 설명
- 위치·기간·운영 정보
- CTA
- 기본 5~7장

#### COLLECTION

- 표지
- 장소별 사진과 짧은 소개
- 전체 목록 요약
- CTA
- 기본 6~10장

#### ROUTE

- 표지
- 장소별 사진과 소개
- 장소 순서와 전체 루트
- CTA
- 기본 6~8장

### 6. Originality Validator

Instagram의 원본성 기준에 맞추기 위해 웹 사진을 그대로 재게시하지 않고 Doripe의 고유한 관점과 정보를 더한다.

- 사진에 장소명·출처·워터마크만 얹은 장은 통과시키지 않는다.
- 각 게시물에는 `Doripe 선정 이유`, `비교·분류`, `추천 상황`, `지도·루트`, `실용 정보`, `순서가 있는 이야기` 중 최소 두 가지를 포함한다.
- 여러 출처의 사진을 하나의 새로운 큐레이션과 정보 구조로 편집한다.
- Doripe의 텍스트와 그래픽을 제거했을 때 원본 게시물과 거의 같아 보이면 실패 처리한다.
- 캡션만 바꾸고 사진을 그대로 다시 올리는 방식은 유의미한 편집으로 인정하지 않는다.

이 원본성 검수는 Instagram 추천 노출을 위한 편집 기준이며 사진 저작권 허가를 대신하지 않는다.

### 7. Typography Validator

모든 문구는 Figma에 입력한 뒤 실제 배치 상태를 확인한다.

1. 기본 글씨 크기로 의미 단위와 단어 경계에 맞춰 줄바꿈한다.
2. 단어 중간에서 끊기거나 조사만 외따로 떨어지는 줄바꿈을 피한다.
3. 영역을 넘으면 의미를 유지하면서 해당 문구만 축약한다.
4. 그래도 넘치면 해당 피드의 해당 텍스트만 기본 크기의 최대 10%까지 줄인다.
5. 최소 크기로도 해결되지 않으면 내보내지 않고 검수 경고를 남긴다.

다른 피드, 다른 장과 원본 템플릿의 글씨 크기는 변경하지 않는다.

### 8. Exporter

검수를 통과한 Figma 프레임을 템플릿에 지정된 Instagram 규격의 PNG로 내보낸다. 한 게시물의 결과는 하나의 폴더로 묶는다.

```text
instagram-content/
├── performance.csv
└── YYYY-MM-DD/
    ├── 01-content-slug/
    │   ├── 01-cover.png
    │   ├── 02-content.png
    │   ├── ...
    │   ├── caption.txt
    │   ├── sources.txt
    │   └── review.txt
    └── 02-content-slug/
        └── ...
```

`sources.txt`에는 사실 자료와 모든 사진의 출처·URL·권리 상태를 넣는다. `review.txt`에는 자동 검수 결과, 권리 미확인 사진, 개인정보 노출 가능성과 사람이 확인할 사항을 넣는다.

`performance.csv`에는 게시 후 사용자가 Instagram Insights에서 확인한 다음 값을 기록한다.

- 게시물 ID와 게시일
- 콘텐츠 유형과 주제
- 조회수·도달
- 비팔로워 도달 비율
- 보내기·저장·좋아요와 각각의 도달 대비 비율
- 프로필 방문·팔로우 전환

다음 일일 실행은 최근 성과에서 보내기 비율과 저장 비율이 높았던 주제·표지 문구·템플릿 특성을 후보 점수에 반영한다.

## 일일 실행 흐름

1. 최근 제작 이력을 읽는다.
2. 기록된 최근 성과를 읽고 공유·저장 성과가 좋았던 특성을 추출한다.
3. 웹에서 콘텐츠 후보를 조사한다.
4. 후보에 점수를 매기고 최대 2개를 선택한다.
5. 사실 정보와 실제 웹 사진을 수집한다.
6. 표지 문구, 장별 정보와 캡션을 작성한다.
7. 해당 Figma 템플릿의 슬롯만 교체한다.
8. 원본성, 줄바꿈, 오버플로, 사진 슬롯과 사실 정보를 검수한다.
9. 검수를 통과한 이미지를 내보낸다.
10. 이미지, 캡션, 출처와 검수 기록을 날짜별 폴더에 저장한다.
11. 사용자가 확인하고 직접 게시한다.

## 실패 처리

- 사실 정보가 부족하면 후보를 교체한다.
- 충분한 고화질 사진이 없으면 후보를 교체한다.
- 최근 콘텐츠와 유사하면 후보를 교체한다.
- 템플릿 슬롯 수가 맞지 않으면 다른 템플릿을 선택하거나 후보를 교체한다.
- 사진 다운로드에 실패하면 같은 후보의 다른 사진을 찾는다.
- Figma 입력 또는 내보내기에 실패하면 원본 템플릿을 변경하지 않고 해당 게시물만 실패 처리한다.
- 기준을 만족하는 후보가 하나뿐이면 품질이 낮은 두 번째 게시물을 억지로 만들지 않고 1개만 생성한다.
- 실패와 미생성 사유는 해당 날짜의 실행 기록에 남긴다.

## 자동 검수

- 모든 필수 `slot:*` 레이어가 존재한다.
- 모든 필수 사진 슬롯이 채워졌다.
- 첫 장에 대상·상황·지역·기간 중 필요한 구체성이 있고 한 가지 약속이 명확하다.
- 각 게시물에 Doripe의 편집·관점 요소가 최소 두 가지 있다.
- 단순 사진 재게시나 출처·워터마크만 추가한 장이 없다.
- 텍스트가 프레임을 벗어나거나 다른 요소를 가리지 않는다.
- 단어 중간의 부자연스러운 줄바꿈이 없다.
- 자동 축소한 텍스트는 기본 크기의 90%보다 작지 않다.
- 내보낸 이미지 수와 크기가 템플릿 설정과 일치한다.
- 장소명, 주소, 날짜와 운영 정보에 근거 URL이 있다.
- 캡션에 확인되지 않은 사실이 없다.
- 사진마다 출처 URL과 권리 상태가 있다.
- 지역·장소 유형·상황 검색어가 표지 또는 캡션에 자연스럽게 포함된다.
- Instagram 위치 태그 제안이 있다.
- 최근 제작 이력과 제목·장소·대표 사진이 중복되지 않는다.
- 게시물에 핵심 메시지와 저장·공유 CTA가 각각 하나다.

## 검증 방법

- 각 템플릿에 짧은 문구, 긴 문구와 긴 한국어 고유명사를 넣어 줄바꿈을 확인한다.
- 세 템플릿별 샘플 콘텐츠를 한 건씩 생성한다.
- 샘플 결과의 Figma 화면과 PNG 내보내기 결과를 비교한다.
- Doripe 그래픽과 문구를 제거했을 때 단순 재게시로 보이는 샘플이 원본성 검수에서 실패하는지 확인한다.
- 사진 누락, 출처 누락, 긴 제목, 행사 종료와 중복 장소 실패 사례를 테스트한다.
- 실제 일일 실행과 같은 조건으로 두 게시물을 생성하는 최종 시험을 한다.

## 완료 기준

- 매 실행에서 기준을 통과한 완성 게시물을 최대 2개 만든다.
- 시스템은 디자인 레이어를 바꾸지 않고 지정된 사진·문구 슬롯만 변경한다.
- 모든 완성 게시물은 정보형 캐러셀이며 Doripe만의 편집·관점 요소를 최소 두 가지 포함한다.
- 결과 폴더에 모든 PNG, `caption.txt`, `sources.txt`, `review.txt`가 있다.
- 문구가 잘리지 않고 단어 중간의 부자연스러운 줄바꿈이 없다.
- 자동 축소는 해당 텍스트에만 적용되고 10%를 넘지 않는다.
- 사실 정보와 사진마다 확인 가능한 출처가 기록된다.
- 표지에는 구체적인 약속이 있고 저장·공유 중 하나의 CTA만 사용한다.
- AI 생성 사진이 포함되지 않는다.
- 사람이 최종 검수한 뒤 직접 게시할 수 있다.

## 주요 위험

- 일반 웹 사진의 출처 표시는 사용 허가를 보장하지 않는다.
- 웹 사진에 피상적 문구만 얹으면 Instagram이 독창적이지 않은 콘텐츠로 판단해 비팔로워 추천 도달을 제한할 수 있다.
- Figma 구조나 레이어 이름을 사람이 변경하면 자동 입력이 실패할 수 있다.
- 행사 정보는 변경·취소될 수 있어 생성 시점마다 재확인이 필요하다.
- 하루 두 게시물의 품질을 유지하려면 중복 이력과 후보 교체 규칙이 중요하다.
- 벤치마크 수치는 여러 계정의 관측 자료이므로 Doripe 성과를 보장하지 않으며 `performance.csv`의 실제 결과로 계속 조정해야 한다.

확인일: 2026-07-14

- [Instagram Ranking Explained](https://about.instagram.com/blog/announcements/instagram-ranking-explained): Feed는 머문 시간, 댓글, 좋아요, 공유, 프로필 방문 가능성을 예측하며 게시물의 빠른 공유·저장·반응과 위치 정보를 신호로 사용한다.
- [Adam Mosseri ranking signals, 2025-01-21](https://www.instagram.com/reel/DFFyRp-pINJ/?hl=en): 연결·비연결 도달 모두 머문 시간, 좋아요와 보내기가 중요하며 비팔로워 도달에는 보내기가 상대적으로 더 중요하다고 설명한다.
- [Instagram Search Explained](https://about.instagram.com/blog/announcements/break-down-how-instagram-search-works): 프로필과 캡션의 관련 키워드·지역, 캡션 안의 관련 해시태그가 검색 노출에 사용된다.
- [Instagram Original Content Guidelines](https://creators.instagram.com/original-content-guidelines): 사진·슬라이드를 주로 그대로 재게시하면 비팔로워 추천 대상에서 제외될 수 있다. 고유한 관점, 정보, 스토리와 창의적 그래픽을 더한 유의미한 편집이 필요하며 단순 출처·워터마크·캡션 변경은 충분하지 않다.
- [Metricool 2026 Instagram Study](https://metricool.com/press-release-instagram-study-2026/): 375,118개 계정의 약 2,436만 게시물 분석에서 캐러셀은 단일 이미지보다 저장이 9배 많았고, 단일 이미지 도달은 전년 대비 21.96% 감소했다.
- [Socialinsider 2026 Instagram Benchmarks](https://www.socialinsider.io/social-media-benchmarks/instagram): 2025년 약 3,500만 게시물 분석에서 캐러셀 참여율은 0.55%, 단일 이미지는 0.35%였다. 계정 규모와 콘텐츠 전략에 따라 결과는 달라질 수 있다.
- [Metricool and HypeAuditor 2025 Instagram Content Playbook](https://metricool.com/press-release-instagram-content-playbook/): 2025년 상반기 약 7억 게시물 분석에서 주 10회 이상 게시한 계정은 더 높은 성장과 연관됐다. 이는 인과관계를 보장하지 않으므로 하루 2개는 품질 기준을 통과할 때만 생성한다.

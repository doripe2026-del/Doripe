# Instagram 감도 사진·실제 앱 화면 디자인

## 목적

Doripe Instagram 캐러셀의 웹 사진 품질을 높이고, 모든 게시물의 마지막 장에서 가짜 UI 대신 실제 Doripe Discover 화면을 보여준다.

이 문서는 기존 `Instagram 콘텐츠 자동화 설계`와 `Instagram 캐러셀 단순화 디자인`을 유지하면서 사진 선별 기준과 마지막 장 표현만 구체화한다. 두 문서가 충돌하면 최신 문서인 이 설계와 캐러셀 단순화 디자인을 우선한다.

## 확정 방향

- 국내에서 촬영된 실제 웹 사진만 사용한다.
- AI 생성·합성 이미지는 사용하지 않는다.
- 사진 분위기는 `밝은 로컬 에디토리얼`로 고정한다.
- 한 캐러셀의 권장 사진 구성은 장소 50%, 사람 25%, 음식·디테일 25%다.
- 모든 콘텐츠의 마지막 장에는 실제 Doripe Discover 화면을 고정 사용한다.
- 마지막 장의 질문 문구만 콘텐츠에 맞춰 바꾼다.
- CTA 버튼과 저장·공유 요청 문구는 넣지 않는다.
- Brain 문서는 이번 작업에서 수정하지 않는다.

## 사진 수집과 선별

### 필수 조건

사진 후보는 다음 조건을 모두 만족해야 한다.

1. 대한민국 안에서 촬영된 장소·행사·음식·풍경 사진이다.
2. 사진 출처 페이지와 원본 URL을 기록할 수 있다.
3. AI 생성 또는 AI 합성으로 표시된 이미지가 아니다.
4. 게시물 주제와 실제 장소가 일치한다.
5. 캐러셀 슬롯에 사용할 수 있는 충분한 해상도를 가진다.
6. 과도한 워터마크, 광고 문구, 앱 UI 캡처가 사진을 가리지 않는다.

재사용 허용이 확인된 사진을 우선한다. 권리 상태가 확인되지 않은 일반 웹 사진은 기존 규칙대로 `not_found`로 기록하고 사람의 게시 전 검수를 요구한다. `restricted` 사진은 사용하지 않는다.

### 감도 점수

필수 조건을 통과한 사진은 다음 기준으로 100점 만점 평가한다.

- 자연광과 밝기: 25점
- 장소의 분위기와 고유성: 25점
- 구도와 명확한 피사체: 20점
- 사람·음식·공간이 전하는 현장감: 15점
- 같은 캐러셀 안의 사진들과 색감 조화: 15점

70점 미만 사진은 사용하지 않는다. 어둡고 탁한 사진, 회색 하늘이 화면 대부분을 차지하는 사진, 장소를 알아보기 어려운 접사, 비슷한 구도의 반복 사진은 우선 제외한다. 야간 행사처럼 어두움이 주제의 핵심일 때만 예외를 허용하고 `review.txt`에 이유를 남긴다.

### 캐러셀 구성

사진 비율은 매 장을 기계적으로 맞추는 절대 규칙이 아니라 한 게시물의 목표 구성이다.

- 장소 50%: 외관, 내부 공간, 풍경, 전시 전경
- 사람 25%: 뒷모습, 걷는 장면, 공간을 경험하는 모습
- 음식·디테일 25%: 음식, 메뉴, 사물, 재료, 빛과 질감

사람 사진은 정면 인물 사진보다 자연스러운 뒷모습이나 환경 속 인물을 우선한다. 얼굴이 식별되면 기존 개인정보 검수 규칙을 적용한다. 같은 장소를 여러 장 쓸 때는 전체 전경, 사람 있는 장면, 세부 장면처럼 구도를 달리한다.

적절한 사진이 부족하면 낮은 품질 사진으로 수량을 채우지 않고 다른 후보 콘텐츠로 교체한다.

## 마지막 브랜드 장

### 화면 구성

- 규격: 기존 Instagram 1080×1350 프레임과 안전영역을 유지한다.
- 배경: 검은색 계열을 사용한다.
- 상단: 콘텐츠 주제에 맞는 질문 한 문장을 중앙 정렬한다.
- 중앙: 실제 Doripe Discover 화면 캡처를 휴대폰 프레임 안에 넣는다.
- 하단: `/Users/cityboy/Desktop/Doripe Assets/icon removed.png`의 Doripe 심볼을 작게 사용한다.
- CTA 버튼, QR 코드, `저장하기`, `공유하기`, `지금 확인하기` 같은 직접 행동 문구는 넣지 않는다.

### 실제 앱 화면 기준

Discover 화면은 Figma 도형으로 다시 그리지 않는다. canonical repository의 실제 Doripe 앱 화면을 고정된 모바일 뷰포트에서 캡처한 PNG를 사용한다.

- 화면: Pinterest형 Discover 피드
- 뷰포트: 기존 앱 기준 모바일 크기 393×852
- 상태: 로딩 스켈레톤이 아닌 사진이 채워진 정상 화면
- 금지: 가짜 카드, 임의 문구, 앱에 없는 버튼, AI 생성 UI
- 갱신: 앱 UI가 바뀌었을 때만 화면 캡처 자산을 다시 만든다.

모든 콘텐츠 유형에서 같은 Discover 화면을 사용한다. 장소·행사·컬렉션·코스마다 화면을 바꾸지 않는다.

### 질문 문구

질문은 콘텐츠별로 바꾸되 한 문장만 사용한다.

- 전시·행사: `이 전시 다음에 갈 장소가 궁금하다면?`
- 단일 장소: `이곳과 함께 갈 장소가 궁금하다면?`
- 장소 모음: `오늘의 취향을 더 발견하고 싶다면?`
- 코스: `새로운 하루의 취향을 찾고 싶다면?`

문구가 안전영역을 넘으면 의미를 유지한 채 먼저 줄인다. 그다음 해당 문구의 글씨만 최대 10%까지 축소한다. 단어 중간 줄바꿈은 허용하지 않는다.

## 자동화 변경

### 사진 단계

1. 기존 Source Collector가 국내 실제 웹 사진을 모은다.
2. 필수 조건 위반 사진을 차단한다.
3. 남은 사진에 감도 점수를 부여한다.
4. 70점 이상 사진으로 장소·사람·음식·디테일 구성을 만든다.
5. 선택 이유와 권리 상태를 `sources.txt`와 `review.txt`에 기록한다.

### Figma 단계

1. 승인된 템플릿의 사진 슬롯에 새 사진만 교체한다.
2. 마지막 브랜드 장은 실제 Discover 화면 캡처 자산을 사용한다.
3. 콘텐츠별 질문 슬롯만 변경한다.
4. Desktop의 Doripe 심볼 asset을 사용한다.
5. 내보내기 전에 안전영역, 줄바꿈, 로고, 앱 화면을 검수한다.

## 검수

### 자동 검수

- 사진마다 국내 촬영 근거, 출처 URL과 권리 상태가 있다.
- AI 생성·합성 이미지가 포함되지 않는다.
- 선택 사진의 감도 점수가 70점 이상이다.
- 같은 캐러셀 안에 지나치게 비슷한 구도가 반복되지 않는다.
- 마지막 장이 검은 배경과 실제 Discover 화면 캡처를 포함한다.
- 마지막 장에 CTA 버튼이나 직접 행동 문구가 없다.
- 질문 문구가 안전영역 안에 있고 단어 중간 줄바꿈이 없다.
- Doripe 심볼이 Desktop asset 원본을 사용한다.

### 사람 검수

- 전체 사진이 밝은 로컬 에디토리얼 분위기로 보이는지 확인한다.
- 사람·음식·장소가 억지로 섞이지 않고 하나의 장소 이야기로 이어지는지 확인한다.
- 앱 화면이 실제 현재 Doripe Discover 화면과 동일한지 확인한다.
- 권리 미확인 사진과 식별 가능한 얼굴을 게시 전에 확인한다.

## 완료 기준

- 두 드라이런 모두 AI 이미지 없이 국내 실제 웹 사진만 사용한다.
- 사진이 밝고 장소의 분위기를 명확하게 보여준다.
- 장소 50%, 사람 25%, 음식·디테일 25%의 권장 구성을 가능한 범위에서 충족한다.
- 모든 마지막 장이 동일한 실제 Discover 화면을 사용한다.
- 마지막 장의 질문만 콘텐츠에 맞게 달라진다.
- CTA가 없다.
- 결과 폴더에 모든 이미지, 캡션, 출처와 검수 기록이 있다.

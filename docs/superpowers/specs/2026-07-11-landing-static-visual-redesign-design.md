# Doripe Landing Static Visual Redesign

## Goal

Create four static Figma concepts for the landing page before rebuilding the motion version. The concepts must make Doripe's product flow immediately understandable while preserving the latest Doripe UI as the visual source of truth.

## Shared Direction

- Use the latest frames from the Doripe Figma file, not the legacy exported UI used in the current landing prototype.
- Keep one complete app screen visible in every concept. Do not replace the product with detached fragments only.
- Use the original high-resolution embedded UI assets at their preserved aspect ratio so interface text and shapes remain sharp.
- Preserve the complete phone-screen aspect ratio. Do not crop the left or right edges.
- Use AI imagery primarily inside place photos, UGC photos, profile photos, and enlarged photographic extensions outside the screen.
- Do not generate UI, buttons, labels, maps, or Korean text with AI.
- Keep the composition static. Motion behavior will be designed only after the four still concepts are approved.

## Concept 1: Hero

- Show the complete latest Discover or Following screen at the center.
- Keep the full screen visible with no side crop.
- Replace feed photography with a coherent high-resolution AI set.
- Extend two or three selected UI photos beyond the phone as larger photographic panels, as though the place imagery is expanding out of the app.
- Connect each expanded panel back to its source photo using restrained Doripe-green focus outlines or leader lines.
- Keep the product screen as the dominant object; external panels explain the content rather than competing with it.

## Concept 2: Social Place Discovery

- Show the complete latest feed screen.
- Fill the feed with candid AI UGC photography of Seoul cafes, restaurants, streets, shops, and people whose faces are not the focal point.
- Add one enlarged selected-place photograph outside the screen.
- Add enlarged friend and curator reactions near that photo, using real Figma-styled profile chips, official badge rules, and short reaction text.
- The final still should clearly communicate that a user discovers a place through content uploaded by friends and curators.

## Concept 3: Nearby Recommendations

- Begin with a large selected restaurant card containing a place photograph.
- Arrange three recommendation cards around it in order: cafe, activity or entertainment, then dessert.
- Every recommendation card must include its own photograph, category, and walking-time cue.
- Add a course-candidate tray below with the selected restaurant and all recommendation cards represented by image thumbnails and visit order.
- Keep the complete latest Doripe screen visible behind or beside the advertising composition.
- Recommendation cards must look like Doripe cards, not generic pills or floating marketing badges.

## Concept 4: Shared Day Folder

- Do not show navigation or turn-by-turn directions.
- Show three photographic place pins converging toward a central day-course folder.
- The folder contains three place thumbnails and a compact course identity.
- Surround the folder with exaggerated but tasteful social reactions: multiple friend avatars, hearts, comments, saves, and one shared-course reaction.
- Keep the complete latest Doripe route screen visible so the folder metaphor remains anchored to the real product.
- The final still should communicate that several places become one shareable day plan that friends react to.

## Figma Deliverable

- Add a clearly named section to the existing Doripe Figma file: `Landing Static Visual Drafts`.
- Create four desktop presentation frames at 1440 x 900.
- Name frames `01 Hero`, `02 Social Discovery`, `03 Nearby Recommendations`, and `04 Shared Day Folder`.
- Place the frames in one horizontal row with consistent spacing.
- Use original Figma screen clones at approximately 393 x 852 visual size.
- Keep all newly created advertising overlays editable.
- The latest Doripe screens are flattened source images in Figma, so preserve those exact high-resolution source images instead of recreating their UI. AI image fills may remain raster assets.

## Acceptance Criteria

- Every frame includes one complete, uncropped latest UI screen.
- No legacy landing-export UI appears.
- UI is sharp at 100% zoom.
- Original UI text, icons, and layout are not regenerated or redrawn.
- AI photography feels like one coherent Doripe campaign.
- Nearby recommendation and course-candidate cards all contain photos.
- The final concept uses pins-to-folder and social reactions, with no navigation motif.
- The product story is understandable from each still without relying on landing-page copy.

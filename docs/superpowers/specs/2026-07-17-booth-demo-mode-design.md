# Doripe 부스 체험 앱 설계

## 목표

데모데이 방문객이 부스 태블릿을 직접 만지며 60초 안에 `사진 발견 → 첫 장소 선택 → 주변 장소 추가 → 코스 완성`을 경험하게 한다.

방문객이 설명 없이 흐름을 끝내고, 감도 높은 사진과 부드러운 전환에서 `오, 괜찮다`는 인상을 받는 것이 성공 기준이다.

## 제품 방향

- 기존 Doripe 앱 화면은 재사용하지 않고 부스 전용 앱을 처음부터 만든다.
- Doripe Brain의 사진 중심 발견과 방문 코스 연결이라는 핵심 가치는 유지한다.
- 기존 코드에서는 로고, 색상, 폰트, 아이콘과 검증된 공통 UI 방식만 선별해 참고한다.
- 앱은 `doripe.kr/demo`에서 제공한다.

## 체험 흐름

1. 시작 화면에서 `60초 코스 만들기`를 누른다.
2. 필터와 태그가 없는 감도 높은 사진 피드를 계속 아래로 둘러본다.
3. 마음에 드는 사진을 눌러 장소 상세를 연다.
4. `이 장소 주변 둘러보기`를 누른다.
5. 첫 장소 카드가 애니메이션으로 작아지며 화면 위쪽 코스 영역으로 이동한다.
6. 아래에 새로운 사진 피드가 이어지고, 방문객이 다음 장소 사진을 하나 이상 고른다.
7. 장소를 하나 이상 고르면 화면 아래에서 `코스 완성하기` 플로팅 버튼이 올라온다.
8. 버튼을 누르면 선택한 장소의 사진과 방문 순서가 있는 `코스 완성!` 화면을 보여준다.
9. `처음부터 다시`를 누르거나 60초 동안 입력이 없으면 시작 화면으로 돌아간다.

## 화면 구성

### 1. 시작

- 큰 장소 사진과 Doripe 원본 로고
- 문구: `오늘의 장소, 사진으로 골라봐요`
- 주 행동: `60초 코스 만들기`

### 2. 사진 피드

- 세로 길이가 다른 사진이 이어지는 2열 피드
- 위치, 필터, 분위기 태그와 사진 위 문구는 표시하지 않는다.
- 사진의 품질과 분위기가 화면의 중심이 된다.
- 사진을 누르면 해당 장소 상세로 이동한다.

### 3. 장소 상세

- 선택한 사진을 크게 보여준다.
- 장소명과 짧은 소개만 표시한다.
- 저장 버튼 대신 `이 장소 주변 둘러보기`를 주 행동으로 사용한다.

### 4. 코스 만들기

- 선택한 첫 장소가 축소되어 화면 위 코스 카드로 이동한다.
- 첫 장소 아래에 주변 장소 사진 피드가 이어진다.
- 사진을 누르면 초록색 테두리와 체크 표시로 선택 상태를 보여준다.
- 두 번 눌러도 같은 장소가 중복 추가되지 않는다.
- 추가 장소가 하나 이상일 때만 `코스 완성하기` 플로팅 버튼을 노출한다.

### 5. 코스 완성

- 제목: `우리의 코스 완성!`
- 선택한 장소의 사진, 이름과 방문 순서
- 짧은 완료 애니메이션
- 주 행동: `처음부터 다시`

## 기술 구조

- 새 폴더: `public/booth-demo/`
- 파일: `index.html`, `styles.css`, `app.js`, `assets/`
- 새 Vercel rewrite: `/demo` → `/booth-demo/index.html`
- 순수 HTML, CSS와 JavaScript만 사용한다.
- 외부 CDN, 실제 API, 로그인과 Supabase는 사용하지 않는다.
- 모든 사진, 로고, 폰트와 아이콘은 로컬 자산으로 제공한다.

## 데이터와 상태

- 장소는 이름, 짧은 소개와 사진만 가진 고정 데모 데이터다.
- 첫 장소와 추가 장소 선택은 브라우저 메모리에서만 관리한다.
- 새로고침하면 항상 시작 화면으로 복구한다.
- 새 체험을 시작할 때 이전 방문객의 선택을 모두 초기화한다.
- 피드는 제한된 사진을 자연스럽게 순환해 계속 이어지는 느낌을 준다.

## 사진 기준

- 감도와 색감이 일관된 장소 사진만 사용한다.
- 권리가 확인된 기존 Doripe 사진을 우선한다.
- 적절한 사진이 부족하면 `imagegen`으로 가상 장소 사진을 만들고 로컬 자산으로 저장한다.
- 외부 웹 사진과 외부 이미지 URL은 최종 앱에 사용하지 않는다.

## 오류 방지와 접근성

- 터치 영역은 최소 52px를 기준으로 한다.
- 빠른 연속 터치에도 화면이 두 번 이동하거나 장소가 중복 추가되지 않게 한다.
- 이미지가 없으면 깨진 이미지 대신 기본 이미지를 보여준다.
- 마지막 입력 후 60초가 지나면 시작 상태로 자동 초기화한다.
- 큰 움직임을 줄이도록 `prefers-reduced-motion` 대체 전환을 제공한다.
- iPad 세로 화면의 safe-area를 반영한다.

## 검증

- 시작부터 코스 완성까지 자동 사용자 여정 테스트를 추가한다.
- 한 장소만 추가한 경우와 여러 장소를 추가한 경우를 검사한다.
- 중복 터치, 새로고침 복구와 자동 초기화를 검사한다.
- 태블릿 세로 크기에서 피드, 플로팅 버튼과 로고가 잘리지 않는지 확인한다.
- 실제 태블릿에서 전체 흐름을 3회 연속 수행한다.
- 기존 저장소 검사와 앱 테스트가 계속 통과하는지 확인한다.

# Doripe 부스 체험 모드 설계

## 목표

데모데이 방문객이 부스 태블릿을 직접 만지며 60초 안에 Doripe의 핵심 가치인 `장소 발견 → 저장 → 코스 완성`을 경험하게 한다.

성공 기준은 방문객이 설명 없이도 흐름을 끝내고, 마지막 코스 완성 화면까지 막힘 없이 도달하는 것이다.

## 체험 흐름

1. 시작 화면에서 `60초 만에 오늘의 코스 만들기`를 누른다.
2. 사진 중심 장소 목록에서 마음에 드는 장소 하나를 선택한다.
3. 장소 상세에서 `저장하기`를 누른다.
4. 데모용 추천 장소 중 두 곳을 추가한다.
5. `코스 완성하기`를 누른다.
6. `연남동 반나절 코스 완성!` 화면에서 장소 사진과 방문 순서를 확인한다.
7. `처음부터 다시`를 누르거나 1분 동안 입력이 없으면 시작 화면으로 돌아간다.

## 구현 범위

- 기존 `/app-preview` 화면, 전환 구조와 로컬 fixture를 재사용한다.
- 부스 전용 체험 진입점을 추가하고 위 흐름에 필요한 화면과 버튼만 노출한다.
- 로그인, 회원가입, 실제 API, Supabase 쓰기, 지도 경로 계산은 사용하지 않는다.
- 장소 데이터와 이미지는 저장소 안의 데모 자산만 사용한다.
- 태블릿 세로 화면을 기준으로 한다.
- 기존 일반 앱 흐름은 변경하지 않는다.

## 화면 구성

### 시작

- Doripe 한 줄 설명: `사진으로 발견하고, 저장해서, 오늘의 코스로`
- 주 행동 버튼: `60초 만에 오늘의 코스 만들기`

### 장소 발견과 저장

- 한 화면에서 비교할 수 있는 사진 중심 장소 목록
- 장소 상세의 주 행동 버튼: `이 장소 저장하기`
- 저장 성공을 즉시 보여주는 짧은 피드백

### 코스 만들기

- 첫 장소는 이미 선택된 상태로 표시한다.
- 추가 후보는 소수만 보여주고 두 곳을 쉽게 고르게 한다.
- 선택 수와 진행 상태를 명확하게 보여준다.

### 완료

- 제목: `연남동 반나절 코스 완성!`
- 세 장소의 사진, 이름과 방문 순서
- 주 행동 버튼: `처음부터 다시`

## 상태와 데이터

- 모든 체험 상태는 브라우저 메모리에서만 관리한다.
- 새 체험을 시작할 때 이전 방문객의 저장·선택 상태를 초기화한다.
- 새로고침해도 항상 정상적인 시작 상태로 복구한다.
- 외부 네트워크 실패가 체험을 막지 않도록 핵심 자산을 로컬에서 제공한다.

## 오류 방지

- 체험 흐름 밖으로 나가는 버튼과 동작하지 않는 기능은 숨긴다.
- 빠른 연속 터치에도 장소가 중복 추가되지 않게 한다.
- 필요한 데모 이미지가 없으면 깨진 이미지 대신 기본 이미지를 보여준다.
- 마지막 입력 후 60초가 지나면 확인 없이 시작 상태로 자동 초기화한다.

## 검증

- 시작부터 완료까지 핵심 흐름 자동 테스트를 추가한다.
- 중복 터치, 새로고침 복구, 자동 초기화를 검사한다.
- 태블릿 세로 크기에서 글자와 버튼이 잘리지 않는지 확인한다.
- 실제 태블릿에서 핵심 흐름을 3회 연속 수행한다.
- 기존 앱 테스트가 계속 통과하는지 확인한다.

## 제외 범위

- 실제 사용자 계정과 데이터 저장
- 실제 위치와 지도 경로
- 공유, 댓글, 팔로우, 좋아요
- 전체 55개 화면 개선
- production 직접 배포

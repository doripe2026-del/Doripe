## 2. Daytrip Korea 피드 분석

### 핵심 특징

| 항목 | 패턴 |
|---|---|
| 콘텐츠 단위 | 장소 1곳 중심 |
| 사진 톤 | 공간의 무드가 먼저 보임. 음식/정보보다 분위기 우선 |
| 문장 톤 | 감성적이지만 과하게 설명하지 않음 |
| 정보량 | 짧음. 장소명, 지역, 핵심 분위기 중심 |
| 소비 방식 | "여기 예쁘다" → 저장 → 나중에 방문 |

### 강점

| 강점 | 설명 |
|---|---|
| 미감 신뢰 | 사진만 봐도 계정의 취향이 보임 |
| 큐레이션 신뢰 | 아무 장소나 올리는 느낌이 적음 |
| 저장 유도 | 길게 설득하지 않아도 사진이 저장 이유가 됨 |
| 브랜드화 | 계정 자체가 하나의 취향 필터처럼 작동 |

### 한계

| 한계 | Doripe 기회 |
|---|---|
| 장소 발견에서 끝남 | Doripe는 저장 후 루트까지 연결 가능 |
| 개인 상황 반영 약함 | Doripe는 데이트, 혼자, 친구, 시간대 기준으로 분기 가능 |
| 지도/동선 연결 약함 | Doripe는 장소를 바로 루트 후보로 만들 수 있음 |

### 한계

| 한계 | Doripe 기회 |

| 개인화 약함 | Doripe는 유저의 선택/저장/루트 기반으로 추천 가능 |

## 4. Doripe가 가져와야 할 피드 구조

### 기본 방향

Doripe 인스타그램은 "장소 추천 계정"이 아니라 "오늘 갈 곳을 고르게 만드는 계정"이어야 한다.

| 콘텐츠 유형 | 목적 | 참고 계정 |
|---|---|---|
| 장소 1곳 소개 | 미감/취향 신뢰 만들기 | Daytrip |
| 오늘의 루트 | Doripe만의 차별점 만들기 | Doripe 고유 |
| 동네별 후보 묶음 | 저장 유도, 탐색 시작점 만들기 | 대한민국 구석구석 |
| 상황별 추천 | 앱 사용 이유 연결 | Doripe 고유 |
| 유저 저장/루트 반응 | 네트워크 효과 만들기 | Doripe 고유 |

## 5. 추천 피드 포맷

### A. 장소 1곳 카드

목적:
- 계정의 미감 신뢰 확보

구성:
1. 대표 사진
2. 장소명
3. 동네
4. 분위기 태그 2개
5. 상황 태그 1개
6. 한 줄 이유

예시:

```text
오브젝트 연남
연남 / 쇼핑·체험

감성적인데 과하지 않고,
데이트 중간에 20분 들르기 좋은 곳.
```

### B. 오늘의 루트

목적:
- Doripe의 핵심 기능을 인스타에서도 보여주기

구성:
1. 루트 제목
2. 장소 3곳
3. 순서
4. 이동 감각
5. 누구에게 맞는지

예시:

```text
연남 조용한 데이트 루트

1. 브런치
2. 소품샵
3. 조용한 카페

걷는 시간은 짧고,
대화할 공간은 많은 루트.
```

### C. 상황별 후보 묶음

목적:
- 앱의 상황 선택 UX와 연결

구성:
1. 상황
2. 후보 4~6곳
3. 각 장소의 짧은 이유

예시:

```text
오늘 연남에서 조용히 만나야 한다면

카페 2곳
식당 2곳
구경할 곳 1곳
```

### D. 저장 유도형 피드

목적:
- 팔로워 행동을 저장으로 유도

구성:
1. 제목은 명확하게
2. 저장할 이유를 바로 제시
3. 너무 많은 설명 금지

예시:

```text
연남에서 실패 확률 낮은 조용한 장소 5곳
```

## 6. 릴스 포맷

### 추천 릴스 구조

| 초 | 내용 |
|---|---|
| 0~2초 | 문제 제시: "연남 가는데 또 어디 갈지 못 정했으면" |
| 2~6초 | 장소 사진 빠르게 3~5장 |
| 6~10초 | 루트 순서 제시 |
| 10~13초 | 상황 태그 제시 |
| 13~15초 | "Doripe에서 바로 루트로 저장" |

### 릴스 제목 후보

```text
연남 데이트, 이렇게 돌면 됨
성수에서 카페만 가기 아까울 때
망원에서 조용한 곳만 고르면
오늘 저녁 루트 1분 컷
```

## 7. 캡션 원칙

### 해야 할 것

| 원칙 | 설명 |
|---|---|
| 첫 문장에 상황을 박기 | "연남에서 조용한 데이트를 해야 한다면" |
| 장소보다 선택 이유를 먼저 쓰기 | "대화하기 좋고, 걷기 부담 없는 루트" |
| 태그를 앱 구조와 맞추기 | 데이트, 혼자, 친구, 조용한, 감성적인 |
| 저장 이유를 명확히 하기 | "나중에 연남 갈 때 쓰기 좋음" |

### 하지 말 것

| 금지 | 이유 |
|---|---|
| "분위기 좋은 핫플" 남발 | 차별점 없음 |
| 장소 정보만 나열 | 네이버지도/블로그와 다를 게 없음 |
| 광고 느낌 강한 문장 | Doripe 신뢰를 깎음 |
| 너무 많은 장소 한 번에 소개 | 저장은 되지만 Doripe의 루트 가치가 흐려짐 |

## 8. Doripe 피드 운영 비율

초기 4주 기준:

| 유형 | 비율 |
|---|---:|
| 장소 1곳 소개 | 40% |
| 루트 추천 | 30% |
| 상황별 후보 묶음 | 20% |
| 앱 기능/사용 예시 | 10% |

주의:
- 앱 기능 소개를 많이 하면 광고처럼 보임.
- 장소 콘텐츠로 신뢰를 만든 뒤, 루트 기능을 자연스럽게 끼워야 함.

## 9. Doripe만의 차별화 포인트

| 기존 계정 | Doripe |
|---|---|
| 여기가 좋다 | 오늘 여기들을 이렇게 가면 좋다 |
| 저장하고 끝 | 저장 후 루트 후보로 연결 |
| 감성 사진 중심 | 감성 + 상황 + 동선 |
| 계정 취향 중심 | 유저 취향과 저장 반응 중심 |

핵심 문장:

```text
Doripe 인스타그램은 장소를 소개하는 계정이 아니라,
오늘 갈 곳을 고르는 과정을 대신 줄여주는 계정이어야 한다.
```

## 10. 바로 실행할 콘텐츠 시리즈

### 시리즈 1. 오늘의 동네 루트

```text
연남 데이트 루트
성수 혼자 걷기 루트
망원 친구랑 가기 루트
```

### 시리즈 2. 실패 확률 낮은 후보

```text
연남 조용한 카페 5곳
성수 밥 먹고 걷기 좋은 곳 5곳
망원에서 대화하기 좋은 술집 5곳
```

### 시리즈 3. 저장한 사람들의 선택

```text
이번 주 Doripe에서 가장 많이 저장된 연남 장소
데이트 루트에 가장 많이 들어간 카페
혼자 가기 태그에서 많이 저장된 곳
```

### 시리즈 4. 앱 연결형

```text
이 장소들 중 3개만 고르면 루트가 완성됩니다
오늘 갈 곳 못 정했으면 오른쪽으로 넘기세요
저장한 장소를 바로 루트로 바꾸는 법
```

## 11. 결론

Doripe는 Daytrip의 미감과 대한민국 구석구석의 대중적 정보성을 둘 다 참고하되, 그대로 따라가면 안 된다.

핵심은 루트다.

인스타에서는 "저장하고 싶은 장소"를 만들고, 앱에서는 "그 저장을 오늘의 루트로 바꾸는 경험"을 줘야 한다.
